# CI/CD Guide for RefaT

## Overview
The **RefaT** repository uses GitHub Actions to provide a full continuous integration and continuous deployment pipeline. The pipeline runs on every push and pull‑request to the `main` branch, performs linting, runs tests on a matrix of Node versions, builds Docker images for the backend (NestJS) and frontend (Next.js), pushes the images to Docker Hub, uploads test reports, and finally deploys the new images to a remote server via SSH.

---

## CI Workflow (`.github/workflows/ci.yml`)
| Job | Description |
|-----|-------------|
| `lint` | Runs `npm run lint` for the backend (`backend-nest`). |
| `test` | Matrix job that runs on Node `18.x` and `20.x`. Installs dependencies, runs `npm run test` for the backend and builds the frontend. |
| `docker-build` | Builds Docker images for backend and frontend using `docker/build-push-action@v4`. Images are **not** pushed by default unless the `push` flag is set to `true`. |
| `upload-artifacts` | Uploads backend test reports and the built Next.js `.next` folder as artifacts for later inspection. |

**Key features**
- **Caching** of npm dependencies (`actions/setup-node@v3` with `cache: npm`).
- **Matrix testing** on two Node versions.
- **Docker image tagging**:
  - `latest`
  - `commit-${{ github.sha }}` for traceability.
- **Secrets required** (see *Secrets* section).

## Deploy Workflow (`.github/workflows/deploy.yml`)
The deploy workflow triggers **only after a successful Docker push**.
1. Checks out the repository.
2. Sets up the SSH agent with the secret `SSH_PRIVATE_KEY`.
3. Connects to the remote server (replace `user@your.server.com` with your actual host).
4. Pulls the newly‑pushed images:
   ```bash
   docker pull ${{ secrets.DOCKERHUB_USERNAME }}/refa-backend:latest
   docker pull ${{ secrets.DOCKERHUB_USERNAME }}/refa-frontend:latest
   ```
5. Runs `docker‑compose -f /path/to/docker-compose.yml up -d --remove-orphans` to restart the containers.

## Required GitHub Secrets
| Secret | Purpose |
|--------|---------|
| `DOCKERHUB_USERNAME` | Docker Hub account name used in image tags. |
| `DOCKERHUB_TOKEN` | Docker Hub access token (write permissions). |
| `SSH_PRIVATE_KEY` | Private SSH key that has access to the deployment server. |
| `SSH_HOST` *(optional)* | Hostname or IP of the server (if you prefer using a variable in the workflow). |
| `SSH_USER` *(optional)* | SSH username (defaults to `user`). |

> **Important:** Add these secrets in **Repository Settings → Secrets → Actions** before the workflows run.

## Environment Variable Reference (`backend-nest/.env.example`)
The backend reads configuration from a `.env` file at the repository root. Below is a quick reference of the most important variables:

| Variable | Example | Description |
|----------|---------|-------------|
| `DATABASE_URL` | `postgresql://user:password@localhost:5432/refat` | Connection string for Prisma. |
| `JWT_SECRET` | `super‑secret‑jwt-key` | Secret used to sign JWT tokens. |
| `NEXT_PUBLIC_API_URL` | `https://api.refat.com` | Base URL that the Next.js frontend uses to call the API. |
| `CORS_ORIGIN` | `https://refat.com` | Allowed origin for CORS. |
| `REDIS_URL` | `redis://localhost:6379` | URL for Redis (queues, caching). |
| `PORT` | `3001` | Port on which the NestJS API runs. |

> Copy `.env.example` to `.env` and fill in the values before running the application locally or in Docker.
